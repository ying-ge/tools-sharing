# AI Tools Directory

说明
----
这是一个社区驱动的“黄页”式仓库，用于集中展示 FigureYa 用户开发的 AI 辅助科研工具。每个工具条目应包含基本功能、使用链接、免费功能与付费功能说明、许可与维护信息，便于科研人员快速对比、试用与引用。

用法提示
-------
1. 在页面上查找：使用表格或目录快速定位工具名 / 分类 / 功能。
2. 阅读条目详情：点击工具链接查看源码、文档或在线演示。
3. 贡献条目：如你是工具作者或维护者，请按贡献指南提交或更新条目（见 CONTRIBUTING.md）。

快速索引
-------
- 类别（例）：数据处理 | 实验设计 | 文献分析 | 可视化 | 模型训练 | 推理/部署
- 标签示例：notebook, web-app, cli, python, r, docker, gpu, cloud
- 状态：active | maintenance | deprecated

如何添加或更新工具条目（贡献指南摘要）
-----------------------------
1. Fork 仓库并新建分支：`add/tool-yourtoolname`
2. 在 `tools/` 目录下添加一个新的 `<slug>.md`（或更新 README 的表格行）。
3. 必填信息：
   - 名称、短描述、repo 链接、维护者、许可、状态
   - 明确列出“免费功能”和“付费功能/计划”（若无付费请注明“无付费功能”）
4. 包含至少一个运行/安装示例（或 demo 链接）。
5. 若工具包含付费/计费项，须声明定价链接或联系方式，并标明是否有免费试用。
6. 提交 PR 并在 PR 描述中说明你是作者或已获授权发布该信息。
7. CI 检查（可选）：仓库可以配置校验脚本确保字段完整性。

条目审核与上架政策（维护者须知）
-----------------------------
- 审核要点：链接可访问性、许可是否允许公开分享、付费功能描述清晰度、隐私合规（是否上传用户数据）。
- 上架后维护：鼓励作者定期更新 last_update 字段；如工具停止维护请标注为 deprecated 并说明替代方案。
- 移除政策：若工具存在安全漏洞且维护者无回应，临时标为“受限”并在 issue 中记录原因，30 天后可移除（需管理员与社区同意）。

常见问题（FAQ）
--------------
Q: 我不是作者但想添加工具信息，怎么做？
A: 可以提交 PR 并在 PR 中标注信息来源与链接。若作者希望更改，请在 issue 中联系。

Q: 如何申诉错误信息或移除条目？
A: 创建 issue，说明证据与请求，仓库维护者会在 7 个工作日内响应。

法律与许可
-----------
- 本仓库仅为目录与链接集合。工具的源码与许可证以各自仓库为准。对第三方工具的使用与付费请阅读对应项目的许可与隐私政策。
- 仓库本身建议使用 CC-BY-4.0（目录和条目文本）以便共享与引用（根据需要更改）。
